# 01 — Архитектура board и фреймовой колонки

## Координатная сетка board

Miro board — бесконечный 2D холст. Origin (0,0) в центре. Координаты могут быть любыми, включая отрицательные.

Для презентации одного урока используется **вертикальная колонка** фреймов, начиная с x=200, y=200. Это не догма — можно сдвинуть. Но если хочешь чтобы Convert to Slides собрал слайды в правильном порядке, фреймы должны быть выстроены сверху вниз последовательно.

## Anatomy of one lesson presentation

```
┌─────────────────────────────────┐  y=200
│                                 │
│   FRAME 1 — Cover (1920×1080)   │
│                                 │
└─────────────────────────────────┘  y=1280
        gap = 100
┌─────────────────────────────────┐  y=1380
│   FRAME 2 — Шаг 1 (1920×1080)   │
└─────────────────────────────────┘  y=2460
        gap = 100
┌─────────────────────────────────┐  y=2560
│   FRAME 3 — Шаг 2 (1920×1080)   │
└─────────────────────────────────┘  y=3640
        ...
```

Y-координаты для 6 фреймов: **200, 1380, 2560, 3740, 4920, 6100**.
Формула: `y_n = 200 + (n−1) × 1180` (1080 height + 100 gap).

X у всех одинаков: `x=200, w=1920`. Это ставит фрейм по центру если board новый.

## Layout внутри одного frame

Frame 1920×1080 разделён вертикально пополам:

```
┌─────────────────────┬─────────────────────┐
│                     │                     │
│   ТЕКСТ             │   ПУСТО             │
│   x=300..1080       │   x=1080..1920      │
│   (relative)        │   (relative)        │
│                     │   (для face-cam     │
│   = x=500..1280     │   при записи)       │
│   на board          │                     │
│                     │                     │
└─────────────────────┴─────────────────────┘
```

**Текстовая зона:** relative `x=300, w=780` = absolute `x=500, w=780`.
**Свободная зона справа:** relative `x=1080..1920` = absolute `x=1280..2120`. ≈840px ширины × 1080 высоты — туда face-cam.

## Vertical anatomy frame (relative coords)

```
y=140    eyebrow  (mono uppercase gold, size 18-22)
y=240    title    (eb_garamond serif white, size 56-80)
y=440    subtitle (plex_sans grey, size 22-24)
y=540+   content  (sections varies by step)
y=900+   pin      (plex_serif italic gold, size 18-22)
y=1020   footer   (cover only, plex_mono dim, size 14)
```

Эти y — **не догма**, это базовая шкала. Подгоняй под количество контента в шаге. Минимальный отступ от верха = 100, от низа = 80.

## Если 5 шагов мало (нужно 6, 7 шагов)

Просто добавляй фреймы дальше по той же формуле:

```
y=7280   FRAME 7 — Шаг 6
y=8460   FRAME 8 — Шаг 7
```

Cover остаётся первым, его «N / TOTAL» footer обновляешь под фактическое число шагов.

## Если воркшоп, не интенсив

Воркшоп EdgeLab — 5-дневная структура (Day 0..4), у каждого дня свои уроки. Структура та же (cover + steps), но:

- Eyebrow: «ДЕНЬ N · УРОК M · MM МИН»
- Footer: `<your-domain>  ·  день N  ·  M / TOTAL`

Контент тащится не из `intensive_lessons`, а из workshop schema (см. workshop skill).

## Сосуществование нескольких уроков на одном board

Допустимо — но клади их колонками **рядом**, не друг под другом. Иначе Convert to Slides соберёт всё подряд.

```
       Урок 1                Урок 2                Урок 3
x=200..2120          x=2400..4320          x=4600..6520
y=200..7280          y=200..7280           y=200..7280
```

Гэп между уроками `Δx ≥ 280`. Чтобы Convert to Slides делать на один урок за раз — выделяй фреймы группой и Convert.
