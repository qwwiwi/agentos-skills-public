# 04 — Редактирование существующего дня

Типичные задачи: обновить текст секции, добавить карточку, поправить заголовок, заменить дату, обновить бейдж. RU+EN всегда меняются парой.

## Базовые правила

1. **Не переписывать с нуля.** День уже имеет согласованную структуру (hero → content sections → CTA). Редактируй блоки, не файл целиком.
2. **RU сначала, EN следом.** Пиши правку в `site/day-N/index.html`, сразу зеркаль в `site/en/day-N/index.html`. Deploy-скрипт упадёт если EN короче RU более чем на 30%.
3. **Никогда не менять `class` без причины.** CSS-классы (`.glass-card`, `.warning-card`, `.cs-*`) одинаковы в RU и EN — это критично для дизайна.
4. **Короткое тире `–`** (U+2013), русские кавычки `«»` в RU, английские `"..."` в EN.

## Частые сценарии

### Обновить текст секции

Читаешь `day-N/index.html`, ищешь нужный блок по заголовку или anchor-id (`#block-0`, `#lesson`), правишь текст внутри `<p>`/`<li>`. Структуру не трогай.

То же в `en/day-N/index.html` — строка с тем же классом/anchor, перевод текста.

### Добавить блок в существующий день

Выбери ближайший семантически подходящий паттерн из текущего дня:

- **Текстовая секция:** `<section class="max-w-3xl mx-auto px-4 sm:px-6 pb-8"> <div class="glass-card"> <h2 class="text-xl font-bold mb-4">...</h2> <p class="text-muted">...</p> </div> </section>`
- **Предупреждение / call-to-action:** `.warning-card` (жёлтый, для «если нашёл проблему — сделай X»)
- **Обязательный раздел (зелёная рамка):** `.claude-cornerstone` — см. `day-2/index.html` как живой образец. Внутри `cs-title` / `cs-sub` / `cs-prompt-box` / `cs-sources-list`.
- **Карточка с иконкой и текстом:** копируй существующую с другой страницы, меняй иконку (inline SVG, stroke `#E8926A` или `#4ade80`) и текст.

### Обновить дату или заголовок

В `<head>`: `<title>`, `<meta name="description">`, `<meta property="og:...">` (если есть).
В `<body>`: `badge-date` блок в hero, H1.

### Перевести бейдж `Скоро` → `Доступно` (день открылся)

В `site/index.html` + `site/en/index.html` карточка этого дня:

| Атрибут | Было | Стало |
|---|---|---|
| Badge class | `badge-soon` | `badge-available` |
| Badge text | «Скоро» / `Soon` | «Доступно» / `Available` |
| Timeline dot class | `timeline-dot-soon` | `timeline-dot-active` |
| Icon wrapper | `bg-accent/10 border border-accent/20` | `bg-green-500/10 border border-green-500/20` |
| Icon stroke | `#E8926A` | `#4ade80` |

**Важно — это одно из любимых граблей user:** иконка и бейдж должны быть одного цвета. Если оставишь оранжевый ключик рядом с зелёным «Доступно» — user сразу заметит (см. `08-learnings.md`).

### Добавить видео-урок или таймкоды

Смотри отдельные гайды — не дублируй руками:
- Видео на странице: `05-add-lesson-video.md`
- Таймкоды из записи эфира: `06-timecodes.md`

## Навигация внутри файла

Структура дня (на примере `day-1/index.html`):

```
<nav>               ← topbar с навигацией и lang-switch
<section hero>      ← badge-date + H1 + подзаголовок
<nav#toc>           ← Оглавление (anchors на секции ниже)
<section#block-0>   ← Видео эфира + таймкоды
<section#lesson>    ← Видео-урок (если есть)
<section#block-N>   ← Тематические блоки эфира по таймкодам
<section#faq>       ← FAQ
<footer>
```

Anchors в TOC должны совпадать с `id` секций — когда добавляешь новую секцию, обнови TOC тоже.

## Проверки перед deploy

```bash
# 1. Сравнить количество строк RU vs EN (вручную, быстро)
wc -l site/day-N/index.html site/en/day-N/index.html

# 2. Прогнать check-only (не деплоит, только валидирует)
bash scripts/deploy.sh --check-only

# 3. Открыть локально в браузере
open site/day-N/index.html
open site/en/day-N/index.html
```

Если `wc -l` показывает разницу > 30% — EN-версия устарела, deploy-скрипт такое блокирует. Сначала синхронизируй.

## Что НЕ делать

- Не дублировать одну и ту же правку в `day-0` / `day-1` / `day-2` / `day-3`, если она логически про один день — каждый файл самостоятелен.
- Не добавлять `<script>` с inline-JS без обсуждения — сайт сейчас полностью статичный, это важно для скорости.
- Не трогать `<script src="https://cdn.tailwindcss.com/...">` версию — зафиксирована, важна для консистентности.
- Не править `site/en/` БЕЗ соответствующей правки `site/`, и наоборот. Паритет — hard rule.
