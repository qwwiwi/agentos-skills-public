# 08 — Learnings: антипаттерны и обратная связь user

Собранная практика: что работает, что бесит user, чего НЕ делать. Обновляется при каждой значимой правке от его фидбека.

## Общий стиль коммуникации

- **Никаких emoji.** Ни в коде, ни в тексте страниц, ни в коммитах.
- **Короткое тире `–`** (U+2013), не `--` и не `—` (длинное тире).
- **Русские кавычки `«»`** в RU-контенте. В EN — обычные `"..."`.
- **Прямо и кратко.** Без преамбул «Привет, сегодня мы поговорим о…». Первой строкой — суть.
- **Бренд:** `EdgeLab` (слитно), не `Edge Lab`. Продукт: `Edge Lab (IEL)`.

## Визуал

### Иконка + бейдж дня = один цвет (hard rule)

На главной (`site/index.html`) у каждого дня есть:
- бейдж (`badge-soon` оранжевый ИЛИ `badge-available` зелёный)
- иконка-квадрат слева от текста (в своей цветной обёртке)

**Иконка ДОЛЖНА совпадать по цвету с бейджем.**

Инцидент 2026-04-20: День 2 открылся (бейдж → `badge-available`, зелёный), но иконка-ключик осталась в оранжевой обёртке. Принц заметил сразу: «Ключик тоже зелёный сделай».

Как исправить при открытии дня:
- `bg-accent/10 border border-accent/20` → `bg-green-500/10 border border-green-500/20`
- `stroke="#E8926A"` → `stroke="#4ade80"`
- Плюс `timeline-dot-soon` → `timeline-dot-active`

См. `02-design-system.md` секция «Иконка дня».

### Никаких ASCII-диаграмм на мобильном

Инцидент 2026-04-21 (Day-1): Архитектура «у тебя на VPS два агента: root + Jarvis + Richard» была нарисована ASCII-рамками в `<pre>`. На десктопе норм, на мобильном `word-break: break-word` ломал рамки в кашу.

**Решение:** responsive CSS-карточки. Desktop 3 колонки через CSS Grid, mobile `@media (max-width: 768px)` → `grid-template-columns: 1fr` (stack). См. `.arch-diagram` / `.arch-agent` классы в `day-1/index.html`.

### Без inline-JS, без новых цветов

- Сайт полностью статичный. `<script>` только для Tailwind CDN. Никакого JavaScript-heavy.
- Все цвета — из палитры `02-design-system.md`. Не импровизируй.

## AI-Native подход

### Тесты для учеников — ОДИН промпт, не список bash-команд

Инцидент 2026-04-20: Day 2 получил раздел «Проверь что CLAUDE.md у тебя настроены правильно». Первая версия была: 4 bash-команды копипастом (`cat ~/.claude/CLAUDE.md`, `ls ~/.claude/rules`, etc.).

Принц: «Проверка теста должна быть максимальная AI Native. Надо отправить, чтобы люди отправили промпт для своего агента, чтобы он сам себя проверил… и приложи источники».

**Правильный паттерн:**

1. **Один копируемый промпт** в `cs-prompt-box` (зелёная рамка), который ученик вставляет в свой Telegram-чат со своим агентом.
2. Агент сам проверяет всё: наличие файлов, содержимое, каскад, корректность хуков, cron-джобов — и отчитывается о каждой проверке с результатом.
3. **Блок «Источники»** (`cs-sources-list`) — 3-5 кликабельных ссылок, которые ученик откроет если агент нашёл расхождения: эталонные репо, installer, гайды по конкретным вопросам.

Почему это работает: у ученика уже настроен агент (цель воркшопа!). Разумный следующий шаг — использовать агента для проверки самого себя, а не заставлять ученика копировать команды руками. Это и есть AI-Native демонстрация.

### Jarvis / Richard — это роли, не имена ботов ученика

В промптах и примерах воркшопа используются имена `Jarvis` (основной агент) и `Richard` (второй, Server Doctor). Это **нашa** (Your project) терминология и архитектурные идентификаторы папок (`~/.claude/`, `claude-gateway`, `claude-richard`).

У ученика **свои имена**. Поэтому в ИНСТРУКЦИЯХ писать нейтрально:

- ❌ «Ричард, у меня в self-diagnostic…»
- ✅ «У меня в самодиагностике основного агента…»

- ❌ «Попроси Jarvis прогнать тест»
- ✅ «Попроси основного агента прогнать тест»

- ❌ «Я (Jarvis) запущен от пользователя edgelab»
- ✅ «Я запущен от пользователя edgelab»

Архитектурные упоминания (`~/.claude/`, `claude-gateway`) можно оставить — это конкретные пути, если ученик ставил через EdgeLab installer. Но в речевых оборотах — нейтрально.

Есть explainer-блок в `day-1/index.html` («Jarvis и Richard — это роли, а не имена ваших ботов»). При добавлении новых промптов — тот же userип.

## Контент и структура

### Не дублируй между днями

Каждый день воркшопа покрывает СВОЮ тему:
- Day 0 — подготовка (ДО эфира)
- Day 1 — База (первый агент)
- Day 2 — Авторский метод (CLAUDE.md каскад, скиллы, тесты)
- Day 3 — Скиллы (углубление)

Если ученику нужна инструкция, которая уже есть в `day-1`, **не дублируй её в day-2** — сошлись на day-1 anchor: `href="/day-1/#block-N"`.

### RU-first, EN — зеркало

Всегда пишешь RU, потом переводишь в EN. Не переводи EN сразу (получится калька). Не пиши EN-first (не соответствует приоритету нашей аудитории).

Структура HTML строка-в-строку одинаковая. Только текст меняется. CSS-классы НЕ менять.

### «Скоро» → «Доступно» workflow

Когда день релизится — см. `03-create-landing.md` секция «Пример: открыть будущий день». Иконку НЕ забыть.

## Операционка

### Репо — источник правды

Не править прод файлы через SSH. Всё через git + `scripts/deploy.sh`. Если потерял локальный клон — `git clone git@github.com:<your-github-org>/<your-workshop-repo>.git`, а не брать файлы с прода через scp.

### Не путать с другими воркшоповыми репо

`<your-github-org>/workshop-edgelab-su` DEPRECATED (2026-04-20). Если случайно зашёл туда — сразу в `online-workshop-edgelab`. Не делать параллельные коммиты.

### `.mov` ломает AssemblyAI

См. `06-timecodes.md`. Ffmpeg в mp3 ПЕРЕД загрузкой — всегда.

## Фидбек-паттерны user

Если user говорит:
- «замёрзла?» / «ты замерзжена?» — я подвисла с ответом, нужно быстро показать прогресс, не извиняться.
- «так не работает» — что-то сломано на проде, проверь URL немедленно, не спорь.
- «не надо» — выкини ту часть, не защищайся.
- «давай лучше [X]» — альтернатива лучше, соглашайся без сопротивления.
- «делай» / «супер» / «пошли» — зелёный свет, двигайся без дополнительных вопросов.
- «ты user?» / «мои агенты» — одобрение предыдущего решения.

Молчание между тулами = баг. Принц смотрит процесс и ждёт статус.
